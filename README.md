## Загальний опис проєкту

**Mine Radiotelescope App** — це спеціалізований програмний симулятор, розроблений у форматі настільного Windows-застосунку. Головною метою проєкту є імітація процесів управління радіотелескопом, захоплення космічних радіосигналів, їх подальшої аналітичної обробки та візуалізації у реальному часі. 

Програмний продукт створено з навчально-дослідницькою метою для відпрацювання навичок проєктування архітектури настільних застосунків, застосування принципів об'єктно-орієнтованого програмування та роботи з багатими графічними інтерфейсами. Проєкт демонструє стійку роботу під час симуляції високих навантажень та повністю відповідає сучасним стандартам розробки прикладного програмного забезпечення.

## Технологічний стек та інструментарій

Додаток побудовано з використанням сучасних інструментів платформи Microsoft, що забезпечує високу продуктивність та стабільність роботи в операційних системах сімейства Windows.
* **Мова програмування:** C#
* **Фреймворк:** .NET 8.0
* **Графічний інтерфейс:** Windows Presentation Foundation (WPF) / XAML
* **Архітектурний підхід:** Розмежування бізнес-логіки та UI з використанням адаптованої моделі ViewModel.
* **Сховище даних:** In-Memory Storage (зберігання даних в оперативній пам'яті).

## Основні функціональні можливості

Симулятор пропонує комплексний підхід до імітації роботи астрономічного обладнання та включає наступні підсистеми:

1. **Модуль автентифікації та сесій.** Надійна система входу користувачів із валідацією облікових даних.
2. **Сканування радіоефіру.** Інтерактивний процес пошуку сигналів за двома ключовими параметрами (полярність та частота) із системою зворотного зв'язку.
3. **Обробка та аналіз даних.** Імітація обчислювального процесу з динамічним відображенням прогресу виконання операцій.
4. **Математична візуалізація.** Побудова графіків захоплених радіохвиль за допомогою генерації випадкових точок у реальному часі.
5. **Динамічна локалізація (i18n).** Можливість миттєвого перемикання мови інтерфейсу (англійська/українська) без необхідності перезавантаження програми.
6. **Тематичне оформлення.** Підтримка динамічного перемикання світлої та темної тем інтерфейсу.

## Архітектура та організація даних

В основу системи покладено принцип інкапсуляції та чіткого розподілу відповідальності між сервісними класами. 

* **Управління користувачами:** Компонент `UserService` функціонує як централізований репозиторій для зберігання зареєстрованих профілів, тоді як `SessionManager` контролює поточний стан авторизації.
* **Акумуляція сигналів:** Аналітичний блок `SignalStore` забезпечує тимчасове зберігання параметрів усіх захоплених радіосигналів протягом активної сесії.
* **Підсистема перекладів:** Багатомовність реалізовано через клас `LocalizationManager`, який використовує складну структуру вкладених словників `Dictionary<string, Dictionary<string, string>>` для забезпечення алгоритмічної ефективності O(1) при пошуку текстових значень.

Усі дані зберігаються виключно в межах поточного сеансу роботи (In-Memory), що робить застосунок легким у розгортанні, позбавляючи його залежності від зовнішніх баз даних чи локальних файлів.
